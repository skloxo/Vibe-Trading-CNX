# PRD v2.4 - A股全市场复盘选股系统

## 概述
两个脚本：
1. **daily_data_maintenance.py** - 数据维护（每个交易日收盘后运行）
2. **daily_dual_screener.py** - 双策略选股（数据维护完成后运行）

数据库路径：`~/.vibe-trading/db/stocks.db`

---

## 一、数据维护脚本 daily_data_maintenance.py

### 1.1 stock_basic 更新
- 同步最新A股列表（新股/退市/改名/ST）
- 数据源：akshare `stock_info_a_code_name()` 或 tushare `stock_basic`
- 每日全量对比，记录变更日志

### 1.2 指数数据（8个指数）
| 指数 | 代码 | akshare API |
|------|------|------------|
| 上证指数 | sh000001 | stock_zh_index_daily_em |
| 深证成指 | sz399001 | 同上 |
| 创业板指 | sz399006 | 同上 |
| 科创50 | sh000688 | 同上 |
| 上证50 | sh000016 | 同上 |
| 中证500 | sh000905 | 同上 |
| 中证1000 | sh000852 | 同上 |
| 北证50 | bj899050 | 同上 |

### 1.3 个股日线增量拉取
- 只拉取最近60天缺失数据
- 并发5线程，每线程间隔0.5秒（避免被ban）
- 数据源：akshare / mootdx / tushare（自动fallback）

### 1.4 技术指标计算
- RSI(14)、MACD(12,26,9)、MA(5,10,20,60)、BOLL(20,2)
- 量比 = 当日成交量 / 过去5日平均成交量
- 5日涨幅、20日涨幅

### 1.5 概念映射更新
- 东方财富概念板块（约500+概念）
- 每只股票可属于多个概念（存储到 stock_concept_mapping 表）
- akshare API: `stock_board_concept_name_em()` + `stock_board_concept_cons_em()`
- 增量更新：只更新有变化的概念

### 1.6 数据质量校验（6项）
1. stock_basic 总数 vs 上一交易日（异常增减告警）
2. 个股日线缺失率（>5%告警）
3. 技术指标空值率
4. 概念映射覆盖率
5. 指数数据完整性
6. 输出校验日志到 `~/.vibe-trading/logs/maintenance_YYYY-MM-DD.log`

## 二、题材热度6维评估（100分制）

| 维度 | 权重 | 说明 |
|------|------|------|
| 板块涨幅 | 20% | >3%=满分 |
| 涨停数量 | 25% | ≥10只=满分，权重最高 |
| 资金净流入 | 20% | >5亿=满分 |
| 量能放大 | 15% | vs5日均量倍数 |
| 连板高度 | 10% | ≥5板=满分 |
| 龙头辨识度 | 10% | 龙头vs板块涨幅差 |

### 热度等级
| 热度分 | 等级 | 标记 |
|--------|------|------|
| ≥90 | 过热 | 🔥🔥🔥 |
| 75-89 | 正热 | 🔥🔥 |
| 60-74 | 升温 | 🔥 |
| 40-59 | 潜力 | 🌱 |
| 20-39 | 冷门 | 🧊 |
| <20 | 冰点 | ❄️ |

## 三、双策略差异化题材匹配

### 策略A: 中线趋势（潜力优先）
```
冰点❄️   → 排除
冷门🧊   → 评分×0.5
潜力🌱   → +15分（最佳买点：资金刚开始关注）
升温🔥   → +10分（舒适区：趋势刚形成）
正热🔥🔥  → +0分（不追高）
过热🔥🔥🔥 → -20分（坚决回避）
```

### 策略B: 龙头情绪（热度优先）
```
冰点❄️   → 排除
冷门🧊   → 排除
潜力🌱   → 评分×0.8
升温🔥   → +5分
正热🔥🔥  → +15分（主战场：最强辨识度）
过热🔥🔥🔥 → +10分（仅保留龙头1只）
```

### 过热退潮机制
- 连续3天过热 → 强制降级为正热

## 四、选股脚本 daily_dual_screener.py

### 4.1 四层筛选
1. **Layer1: 基本面初筛** - 剔除ST、停牌、上市<60天、日均成交<5000万
2. **Layer2: 技术面筛选** - RSI/MACD/MA/量比条件
3. **Layer3: 题材匹配** - 根据策略A/B差异化评分
4. **Layer4: 综合排序** - 按综合评分排序

### 4.2 输出
- 每组精选10只（TOP_PICKS=10）
- 操作池5只（TOP_OPERATE=5）
- 操作池含买点/止损(-7%)/目标(+10%)
- 同题材最多2只（避免集中）
- 操作池同题材最多1只
- 报告输出到 `~/.vibe-trading/reports/daily/YYYY-MM-DD_dual_picks.md`

### 4.3 选股理由格式
```
属于[题材名] 热度等级; RSI XX.X 状态; MA5>MA20 多头/空头; 量比X.X 放量/缩量
```

### 4.4 大盘环境评分（影响仓位建议）
- 涨跌比（涨家数/总数）30分
- 涨停数 20分
- 量能（vs5日均量）20分
- 北向资金净流入 15分
- 指数趋势 15分
- 总分100 → ⭐⭐⭐强势/⭐⭐偏弱/⭐弱势

## 五、执行流程
```
16:00 python3 daily_data_maintenance.py   # 数据维护
18:00 python3 daily_dual_screener.py      # 选股
输出: ~/.vibe-trading/reports/daily/YYYY-MM-DD_dual_picks.md
```

## 六、技术约束
- Python 3.12
- 依赖：akshare, mootdx, tushare, pandas, sqlite3
- 数据库：SQLite，路径 ~/.vibe-trading/db/stocks.db
- 并发：ThreadPoolExecutor(max_workers=5)
- 错误处理：每个数据源失败自动fallback到下一个
- 日志：print输出 + 文件日志
